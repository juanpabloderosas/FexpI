2. Matrices¶

Las matrices en R son idénticas (en cuanto arreglo) a las del álgebra: un arreglo de números en filas y columnas. Para escribir una matriz, deberemos escribir los elementos y el número de filas (nrow) y columnas (ncol):

Valores especiales¶

Si bien, en principio, es posible utilizar cualquier nombre para una variable, hay combinaciones de caracteres especiales, que refieren a funciones o a palabras reservadas por el lenguaje para funciones especı́ficas. Algunas de ellas son:

  • TRUE y FALSE son los valores lógicos, reemplazables por T y F.
  • NA quiere decir Not Available (no disponible), y es una palabra reservada.
  • Inf es el símbolo para indicar infinito, tiene semejanzas con la matemática, pero en este tipo de lenguajes se obtiene como resultado de saturar la cantidad de números posibles...sumar números muy grandes puede entregar este símbolo. 1/0, -1/0, etc. dan Inf.
  • NaN (Not a Number) indica que No es un Número....y se obtiene de hacer Inf * 0, Inf/Inf, etc...

Algoritmo - Una discusión torpe y rápida¶

“Un algoritmo es una conjunto sistemático de procedimientos –en un número finito de pasos– que responden a una pregunta o solucionan un problema. El nombre deriva de la publicación latina Algoritmi de numero Indorum, (“al-Khwuarizmi: Sobre el arte hindú para calcular”) que es una traducción del libro escrito por el matemático musulmán del S. IX al-Khwarizmi"

Entonces, un algoritmo es una receta, sin ambiguedades.

Algoritmo en R¶

Para hacer un algoritmo en R:

  • Generar un archivo de texto (generalmente es de extensión *.R), conteniendo líneas de código (instrucciones del lenguaje). Esto se ve arriba a la izquierda en R Studio
  • Luego utilizando el botón “Source”, R ejecutará secuencialmente (de la primera a la última) nuestro código. Cada línea debe estar libre de errores de redacción (no se va a interpretar nada que esté mal escrito), por lo que puede resultar una tarea, al principio, muy engorrosa.

  • Para ejecutar una lı́nea o una parte de un código, es posible:

    • a) seleccionar una parte del código y presionar el botón Run, al lado de Source

    • b) poner el cursor en una lı́nea y apretar Ctrl + ENTER. Ejecutar pedazos o lı́neas de un código es importante para hacer debugging (desenbichado, en inglés), es decir, en criollo, encontrar los posibles errores del código.

Ejemplos de Algoritmos¶

1. Ángulo entre vectores (un ejemplo simple )¶

Como ejemplo simple ponemos el cálculo del ángulo entre dos vectores. Un diagrama de flujo es un gráfico que conecta "bloques" de un código.

Por ejemplo, ¿qué necesitamos hacer para calcular el ángulo entre dos vectores?

Sabemos que, dados dos vectores $\vec{a}$ y $\vec{b}$, el ángulo $\alpha$ entre ellos estará dado por:

$$ \alpha = cos^{-1}\Bigg(\frac{\vec{a} \cdot \vec{b} } {|\vec{a}| |\vec{b}|} \Bigg) $$

Entonces, para saber el ángulo $\alpha$, necesitamos:

  1. Dos vectores de entrada (Input de nuestro código) $\Rightarrow$ 2. Calcular el producto punto entre los vectores $\Rightarrow$ 3. Calcular el módulo de cada vector$\Rightarrow$4. Dividir el producto punto por el producto de los módulos$\Rightarrow$ 5. Calcular al arco coseno del resultado anterior $\Rightarrow$ 6. Expresar el resultado en radianes y grados sexagesimales
In [56]:
# 1. Dos vectores de entrada
a <- c(1,0,1); b <- c(1,0,-1)
#2. Calcular el producto punto
prodpunto <- sum(a*b)
#3. Modulos
mod.a <- sqrt(sum(a^2)); mod.b <- sqrt(sum(b^2));
#4. Dividir prodpunto por (mod.a * mod.b)
p <- prodpunto/(mod.a*mod.b)
#5. calcular el arco coseno de p
alpha <- acos(p)
#6. Expresar el resultado en radianes y grados sexagesimales
cat(bold$blue("El ángulo entre a  y b  es de", alpha, "rad"))
cat(bold$red("\nEl ángulo entre a y b es de", alpha*180/pi, "°"))

El ángulo entre a y b es de 90 °

2. Superficie plana de una mano.¶

La idea es estimar la superficie de una mano (la mía, que es una mano morena).

Para esto:

  1. Saco una foto de mi mano sobre una hoja A4 (29.7 x 21 cm)
  2. Usamos el contraste (distinguimos negro de blanco) para seleccionar píxeles.
  3. Sabiendo el número de píxeles de la mano respecto del total, calculamos la superficie plana de mi mano.
